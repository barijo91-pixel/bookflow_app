* 사업 배경 및 시스템 개요
프로젝트명 : BookFlow
- 현재 영어 교재(유치·초등 중심) 유통 시장은 대부분 전화, 카카오톡, 수작업으로 이루어지고 있어 비효율이 매우 높습니다. 
  이러한 Pain Point를 해결하는 도서 도매 유통 전문 플랫폼으로, 총판·영업자·학원·학부모를 연결하는 올인원 솔루션을 목표로 합니다.

* 코드 테이블 구성
  그룹코드, 세부이름, 순서, 사용유무
  시스템 관리자가 등록, 수정, 삭제할 수 있게 구성

  생성해야할 코드테이블
   사용자 그룹, 거래처 상태, 도서상태, 은행코드, 출판사, 과목분류, 학습대상, 난이도, 공급가율

    

* 주요 사용자 역할 및 권한
- 시스템 관리자 
- 총판 (출판사/도매 공급자): 플랫폼 최상위 공급자
- 영업자 (1인 사입자/프리랜서): 플랫폼 핵심 사용자 (관리할 학원을 내 소속 하위에 등록하고, 할인율은 등록)
- 학원 (원장/교재 담당자): B2B 구매자이자 B2C 중개자
                                                                                                - 학부모/학생: 앱 설치 없이 웹뷰로 이용 ----- 주문은 어느 소속으로 되나(등록시 소속학원을 선택?)
user table 주요항목 
       이름, 아이디(이메일 형식), 비번, 지역(시, 군, 구 동 단위로 지역 선택할 수 있게 데이타 입력도 시, 군, 구 동 단위 동일하게), 연락처, 등록일시, 수정일시 
       회원 등록 시 인증번호 이메일로 전송하고 일치히면 등록


* 핵심 서비스 Flow (사용자 설명 기반 정리)
 
1. 영업자가 학원 거래처를 등록하고 계약 정보를 입력합니다.
    vendor table 주요항목 
     거래처명, 대표자, 거래처 구분, 상태(정상, 일시정지, 거래종료) , 할인율, 사업자번호, 업태, 종목, 모바일 , 일반전화, 주소, 은행명, 계좌번호, 예금주, 기타사항, 등록일시, 수정일시


2. 학원은 앱에서 도서 목록을 검색·열람하고, 계약된 할인율이 자동 적용된 상태로 교재를 주문합니다. (바코드 스캔 지원)
     Book table 주요항목
     출판사, 과목분류, 교재명, 부제명, 시리즈명, 학습대상, 난이도 레벨, 정가, 할인율, 재고수량, 도서상태(판매중, 품절, 절판, 출간예정),  저자, 규격, isbn, 출간일, 판쇄정보, 표지경로, 첨부파일 경로, 등록일시, 수정일시


3. 주문내용이 담당 영업자 앱으로 실시간 전송되며, 영업자는 검토 후 스와이프 등으로 확정 → 총판(등록된 카카오 아이디)에게 전달합니다.
   order table 주요항목
   주문일시, 영업자, 거래처명, 출판사, 교재명, 수량, 정가, 할인율, 거래처 주소, 카카오id


4. 총판은 주문을 접수해 패킹·출고를 처리하고, 배송 현황을 자동 업데이트합니다.


5. B2C 확장: 학원(또는 영업자)이 클래스별 교재 목록을 편성 → 학부모에게 공유 링크 발송 → 학부모는 웹뷰에서 부분 선택·결제 (도서정가제 준수, 최대 10% 할인).

6. 학원 마진 및 배송비 자동 정산 기능 탑재.
   

* 학부모 앱 설치 여부: 불필요 (웹뷰 + 카카오페이/링크 결제). 학원과 영업자는 네이티브 앱 필수.
- ERP 연동: 초기에는 주문서 PDF·카카오·문자·이메일 자동 발송으로 충분. 이후 API 연동으로 고도화 가능.
- 이 구조를 통해 총판은 영업 인력 없이도 프리랜서 영업자 네트워크 운영, 영업자는 마진·매출·수금 관리 효율화, 
   학원은 견적·수작업 최소화, 학부모는 편리한 개별 주문이 가능해집니다.
